# Lemondory Game Server - 아키텍처 문서

## 🏗️ 전체 아키텍처 개요

### **핵심 설계 원칙**
- **ECS (Entity-Component-System)**: 데이터 지향 설계
- **스레드별 맵 분리**: 맵당 독립 스레드로 동시성 보장
- **네트워크 계층화**: ASIO 기반 비동기 I/O
- **모듈화**: 재사용 가능한 컴포넌트 구조

### **아키텍처 다이어그램**
```
┌─────────────────────────────────────────────────────────────┐
│                    Lemondory Game Server                    │
├─────────────────────────────────────────────────────────────┤
│  Network Layer (ASIO)                                       │
│  ├─ asio_server.cpp    (서버 소켓 관리)                        │
│  ├─ asio_channel.cpp   (클라이언트 연결)                        │
│  └─ frame_codec.hpp    (프로토콜 인코딩)                        │
├─────────────────────────────────────────────────────────────┤
│  Game Layer (ECS + Threading)                               │
│  ├─ game_server.cpp    (메인 게임 서버)                        │
│  ├─ game_service.cpp   (ECS 엔진)                            │
│  └─ threading/         (스레드 관리)                          │
│     ├─ main_thread_manager.cpp                             │
│     └─ map_thread_manager.cpp                              │
├─────────────────────────────────────────────────────────────┤
│  ECS Engine                                                │
│  ├─ entity.hpp         (엔티티 관리)                          │
│  ├─ component.hpp       (컴포넌트 시스템)                      │
│  ├─ system.hpp          (시스템 로직)                         │
│  └─ systems/            (게임 로직)                          │
│     ├─ movement_system.cpp                                 │
│     ├─ ai_system.cpp                                       │
│     ├─ combat_system.cpp                                  │
│     └─ map_system.cpp                                     │
└─────────────────────────────────────────────────────────────┘
```

## 🎯 중점 검토 영역

### **1. 핵심 네트워크 레이어** ⭐⭐⭐
**파일**: `network/asio_server.cpp`, `network/asio_channel.cpp`
- **역할**: 클라이언트 연결 관리, 패킷 송수신
- **중요도**: 최고 (서버의 핵심)
- **검토 포인트**:
  - 연결 안정성 (재연결, 타임아웃)
  - 패킷 순서 보장
  - 메모리 누수 방지
  - 동시 연결 처리

### **2. ECS 엔진** ⭐⭐⭐
**파일**: `game/entity.hpp`, `game/component.hpp`, `game/system.hpp`
- **역할**: 게임 오브젝트 관리, 컴포넌트 시스템
- **중요도**: 최고 (게임 로직의 핵심)
- **검토 포인트**:
  - Entity 생성/삭제 성능
  - Component 메모리 관리
  - System 업데이트 순서
  - 스레드 안전성

### **3. 스레딩 시스템** ⭐⭐
**파일**: `game/threading/main_thread_manager.cpp`, `game/threading/map_thread_manager.cpp`
- **역할**: 맵별 독립 스레드, 작업 분배
- **중요도**: 높음 (성능과 안정성)
- **검토 포인트**:
  - 스레드 간 통신
  - 데드락 방지
  - 작업 큐 관리
  - 맵 이동 처리

### **4. 게임 서버 메인** ⭐⭐
**파일**: `game/game_server.cpp`, `game/main.cpp`
- **역할**: 전체 서버 조율, 초기화
- **중요도**: 높음 (서버 진입점)
- **검토 포인트**:
  - 초기화 순서
  - 에러 처리
  - 리소스 정리
  - 설정 관리

### **5. 게임 로직 시스템** ⭐
**파일**: `game/systems/` 디렉토리
- **역할**: 실제 게임 로직 구현
- **중요도**: 중간 (컨텐츠 의존)
- **검토 포인트**:
  - MovementSystem (이동 로직)
  - CombatSystem (전투 로직)
  - AISystem (AI 로직)
  - MapSystem (맵 관리)

## 🔧 기술 스택

### **네트워크**
- **ASIO**: 비동기 I/O, 크로스 플랫폼
- **프로토콜**: 커스텀 프레임 기반 (CRC32 + 시퀀스)
- **연결 관리**: 채널 기반 연결 추상화

### **게임 엔진**
- **ECS 패턴**: Entity-Component-System
- **메모리 관리**: 스마트 포인터 + 메모리 풀
- **스레딩**: 맵별 독립 스레드 + 메인 스레드

### **빌드 시스템**
- **CMake**: 크로스 플랫폼 빌드
- **C++17**: 모던 C++ 기능 활용
- **외부 라이브러리**: ASIO, spdlog (third_party)

## 📊 성능 특성

### **동시성 모델**
- **메인 스레드**: 네트워크 I/O + 글로벌 시스템
- **맵 스레드**: 맵별 게임 로직 (독립 실행)
- **통신**: 큐 기반 비동기 메시지 전달

### **메모리 관리**
- **Entity**: ID 기반 가벼운 참조
- **Component**: 타입별 스토어 + 메모리 풀
- **네트워크**: 동적 버퍼 + 재사용

### **확장성**
- **수평 확장**: 맵 스레드 추가로 용량 증가
- **수직 확장**: 컴포넌트/시스템 추가로 기능 확장
- **모듈화**: 게임 타입별 커스터마이징 가능

## 🚀 실행 및 테스트

### **빌드 명령어**
```bash
# 빌드
cmake -B build && cd build && make -j4

# 서버 실행
./game_server

# 테스트 실행
./run_tests.sh
```
